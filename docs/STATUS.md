## 已知阻塞

### 1. `secUid` 识别错误

- 当前 CloakBrowser 扫码登录路径可能把错误账号写入数据库
- 这会导致归档内容来自错误的抖音账号
- 该问题尚未根治，当前不适合把扫码路径视为稳定入口

### 2. 验证码 / 风控中间页

- 扫码流程遇到滑块、短信或风险验证时，当前实现缺乏完整处理
- 前端会出现卡在等待中的体验

### 3. 插件链路尚未完全闭环

- 扩展骨架、桥接接口、推送机制都已存在
- 但仍缺少针对真实场景的系统联调与稳定性验证

## 当前建议的开发顺序

1. 先保持数据库与本地测试环境干净，避免被错误账号数据干扰
2. 优先把“手动 Cookie 导入 → 归档 → 下载 → 本地播放”这条链路压稳
3. 然后做 M3 的内容类型扩展和全文搜索
4. 最后再回头处理扫码登录和插件全链路增强

## 不建议现在做的事

- 不建议现在继续深挖扫码登录，除非就是为了修 `secUid` 和验证码问题
- 不建议把管理后台、创作者平台当作当前主线
- 不建议引入额外构建系统去重做浏览器扩展
