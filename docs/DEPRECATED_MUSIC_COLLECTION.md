# 收藏音乐接口下线说明

## 结论

DoLike 已停止维护和暴露“抖音收藏音乐”采集能力。

原因不是实现细节不够，而是抖音网页端相关接口已经进入不可稳定依赖状态：

- 常见接口返回 `404` 或路径失效
- 即使路径可达，也频繁返回“参数不合法”或风控错误
- 页面 SSR / React Fiber / DOM 降级数据并不稳定，无法作为可靠主链路
- 为了维持这条能力，需要持续跟进页面签名、网关路径和风控策略，维护成本过高

## 本次调整

- 扩展弹窗移除了“推送收藏音乐”按钮
- 抖音页面浮层移除了“推送收藏音乐”入口
- 扩展 background 不再接受 `COLLECT_MUSIC` 推送
- 后端归档主流程不再默认跑 `COLLECT_MUSIC` fetcher

## 保留内容

- 历史数据库中的 `COLLECT_MUSIC` / `MUSIC` 数据不做破坏性删除
- 代码中与本地音频、历史兼容相关的类型和部分实现暂时保留，避免影响已归档数据读取
- 页面采集器中少量音乐相关调试代码保留为历史排查参考，但不再对外暴露

## 后续建议

- 不再继续投入抖音收藏音乐网页 API 的修复
- 如果后续需要“音频”能力，优先走本地视频提取音频、或用户主动上传/导入本地文件
- 如果抖音未来重新开放稳定接口，再单独评估是否恢复，而不是在当前链路上继续打补丁
